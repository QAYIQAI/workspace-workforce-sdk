# Code Standards

## Algemene standaarden

- TypeScript waar mogelijk
- Duidelijke naamgeving
- Kleine componenten
- Services per domein
- Validatie met Zod
- Geen duplicatie van businesslogica
- Geen secrets in code
- Geen ongecontroleerde any-types
- Errors centraal afhandelen

## Naming

- Entiteiten in Engels
- UI-labels in Nederlands
- Database tabellen snake_case
- TypeScript types PascalCase
- Functions camelCase
- Constants UPPER_SNAKE_CASE

## Git

- main voor productie
- develop voor integratie
- feature branches
- Pull requests
- Beschrijvende commits
- Migraties apart controleren
