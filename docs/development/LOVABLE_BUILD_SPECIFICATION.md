# Lovable Build Specification

Deze specificatie beschrijft hoe Workspace Workforce in Lovable gebouwd moet worden.

## Buildprincipes

- Bouw gefaseerd.
- Gebruik Supabase als database.
- Gebruik Auth.
- Gebruik RLS.
- Houd UI professioneel.
- Maak mobiel vanaf het begin goed.
- Houd componenten herbruikbaar.
- Voorkom dat Lovable eigen datamodel verzint buiten de SDK.

## Eerste fase

- Login
- Dashboard
- Layout
- Sidebar
- Mobiele drawer
- Rollen en rechten
- Instellingen
- Werknemers
- Opdrachtgevers
- Locaties

## Tweede fase

- Contracten
- Vaste planning
- Dagplanning
- Urenregistratie
- Weekstaten

## Derde fase

- Documenten
- Loonstroken
- Verzendlog
- Boekhoudingsexport

## Vierde fase

- Automatisering
- Make/n8n
- AI-assistent
- Klantportaal
- Medewerkerportaal

## UI-eisen

- Premium SaaS-look
- Dark mode correct
- Tabellen mobiel swipebaar
- Geen te hoge rijen
- Leesbare tekst
- Duidelijke statusbadges
