# AI Development Standards

AI-code agents moeten bouwen volgens deze standaarden.

## Algemene regels

- Werk modulair.
- Breek bestaande functionaliteit niet.
- Respecteer rollen en rechten.
- Voeg server-side validatie toe.
- Houd mobiel gebruik werkend.
- Voeg auditlog toe bij kritieke mutaties.
- Geen gevoelige data in frontend.
- Geen secrets hardcoden.
- Geen publieke documentlinks voor personeelsbestanden.

## Promptregels voor AI-builds

Elke buildprompt bevat:

- Doel
- Scope
- Niet wijzigen
- Bestanden
- Datamodel
- UI-gedrag
- Rechten
- Validaties
- Testscenario’s
- Acceptatiecriteria

## Acceptatiecriteria

Een AI-build is pas klaar wanneer:

- Desktop werkt
- Mobiel werkt
- Dark mode werkt
- Rechten werken
- Validaties werken
- Fouten duidelijk zijn
- Geen console-errors zichtbaar zijn
- Bestaande flows niet kapot zijn
